# COBOLアカウント管理システム - テストプラン

## 概要

このテストプランは、現在のCOBOLアプリケーションのビジネスロジックと実装を検証するための包括的なテストケースを定義します。これらのテストケースは、将来のNode.jsアプリケーションへの移行時に参考とりできる仕様として機能します。

---

## テスト対象範囲

- **Main Program (main.cob)**: メニュー処理、ユーザー入力の受け取り
- **Operations Program (operations.cob)**: 口座操作ロジック（TOTAL、CREDIT、DEBIT）
- **Data Program (data.cob)**: データの読み書き、永続化

## テスト実行手順

### 前提条件
- COBOLコンパイラ（GnuCOBOL等）がインストール済み
- ソースコード：main.cob、operations.cob、data.cob
- コンパイルコマンド：`cobc -x src/cobol/main.cob src/cobol/operations.cob src/cobol/data.cob -o accountsystem`

### テスト実行方法
1. アプリケーションをコンパイル
2. `./accountsystem` でアプリケーションを実行
3. メニュー画面が表示されたら、テストケースに従い入力値を入力
4. 各テストケースの「期待される結果」と「実際の結果」を比較
5. テストプラン上の「実際の結果」「ステータス」「コメント」を記入

### ステータス判定基準
- **合格（PASS）**：期待される結果と実際の結果が完全に一致
- **不合格（FAIL）**：期待される結果と実際の結果が異なる、またはシステムエラーが発生

- 各テスト実行後、テストプランの「実際の結果」欄を記入してください
- 不合格（FAIL）の場合、「コメント」欄に詳細を記入してください

## Node.js移行への参考

このテストプランは、将来のNode.js移行時に以下の点を検証するためのベースラインとして機能します：

- **ビジネスロジックの完全性**：Node.jsアプリケーションが同じビジネスロジックを実装していることを確認
- **データ永続化**：データベース統合後も同じビジネスルールが適用されることを検証
- **入力値の妥当性チェック**：Node.jsアプリケーションが同じレベルのバリデーションを実装していることを確認
- **エラーハンドリング**：エラーメッセージと動作が一貫していることを確認
